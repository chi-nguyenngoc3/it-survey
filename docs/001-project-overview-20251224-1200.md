# ITSM Survey Application - Project Overview

**Document ID**: 001  
**Version**: 20251224-1200  
**Status**: Complete

## Executive Summary

This document provides an overview of the ITSM Survey Application, a production-ready web application designed for IT Service Management data collection in educational institutions. The application has been built following the implementation plan outlined in `ITSM_Implementation_Plan.md`.

## Project Objectives

1. Transform standalone HTML form into a production-ready web application
2. Implement database persistence using Supabase
3. Add multi-language support (Vietnamese default, English available)
4. Apply corporate styling inspired by NHG.vn design
5. Implement auto-save functionality
6. Support data export capabilities

## Technology Stack

| Component | Technology |
|-----------|------------|
| Framework | Next.js 14 (App Router) |
| Language | TypeScript (strict mode) |
| Database | Supabase (PostgreSQL) |
| Styling | Tailwind CSS + shadcn/ui |
| i18n | next-intl |
| Validation | Zod schemas |
| State Management | React hooks |

## Architecture

### Application Layers

1. **Presentation Layer**
   - React components with TypeScript
   - shadcn/ui component library
   - Tailwind CSS for styling
   - next-intl for internationalization

2. **Application Layer**
   - Next.js App Router
   - API Routes for backend operations
   - Middleware for locale detection

3. **Data Layer**
   - Supabase PostgreSQL database
   - Zod validation schemas
   - Local storage fallback

### Key Features Implemented

- ✅ 11 comprehensive survey sections
- ✅ Multi-language support (Vietnamese/English)
- ✅ Auto-save every 30 seconds
- ✅ Progress tracking with visual indicators
- ✅ Data export as JSON
- ✅ Corporate NHG.vn color scheme
- ✅ Responsive mobile-first design
- ✅ Form validation with Zod

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/survey` | List all surveys |
| POST | `/api/survey` | Create new survey |
| GET | `/api/survey/[id]` | Get survey by ID |
| PATCH | `/api/survey/[id]` | Update survey |
| DELETE | `/api/survey/[id]` | Delete survey |

## Deployment

### Development

```bash
npm run dev
```

### Production Build

```bash
npm run build
npm run start
```

### Environment Variables

```env
NEXT_PUBLIC_SUPABASE_URL=your-url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-key
SUPABASE_SERVICE_ROLE_KEY=your-service-key
```
